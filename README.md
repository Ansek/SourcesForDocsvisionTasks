# О курсе:
Курс включает в себя теорию и практику. В ходе обучения мы пройдем полный цикл внедрения решений на базе платформы Docsvision:
- развернем платформу на ОС AstraLinux;
- настроим PostgreSQL для работы с платформой;
- научимся находить медленные запросы PostgreSQL и оптимизировать их;
- напишем модули расширения для платформы на С#;
- реализуем контроллеры ASP.Net WebAPI;
- познакомимся с DI контейнерами;
- научимся взаимодействовать с Rest API из браузера при помощи TypeScript;
- напишем свой элемент управления на React для Docsvision WebClient.

# Кратко о заданиях:
1. Установка компонентов Docsvision на виртуальных машинах Astra Linux 1.7 и Windows Server 2019.
2. Заполнение данных об организациях и сотрудниках. Создание карточки заявки на командировку.
3. Реализация обработчиков событий (TypeScript) c целью проверки содержимого заполненных полей.
4. Работа с конструктором состояний (настройка жизненного цикла документа) и ролей. Модификация Task3 для вывода данных из справочника (информация о городах).
5. Формирование папок с целью фильтрации документов по условиям.
6. Реализация консольного приложения (C#) для автоматической генерации большого количества заявок на командировки с целью тестирования производительности.
7. Реализация сервисов (C#) для запроса пользовательских метаданных документа с целью автозаполнения полей при изменении связанных данных (TypeScript).
8. Реализация плагина DataGrid (C#) для получения данных из хранимой функции PostgreSQL.
9. Реализация парсинга CSV-файла (C#) для заполнения 3-х связанных таблиц БД. Написанное приложение преобразует данные в JSON и отправляет результат в хранимую процедуру (PL/pgSQL) для дальнейшего распределения.
10. Реализация ролевой функции (PL/pgSQL) для ограничения доступа к документам. Задача больше касалась оптимизации запроса используя EXPLAIN ANALYZE.
11. Реализация пользовательского элемента управления (C#, TypeScript), который запрашивает информацию о рейсах с сервиса https://api.travelpayouts.com/ (Task3 - графическая часть, Task7 - обращение к сервису, Task11 - регистрация компонента в конструкторе).
